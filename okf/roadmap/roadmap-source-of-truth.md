# Roadmap Source of Truth

The canonical DiscWeave roadmap is the GitHub Project:

https://github.com/users/Fredoq/projects/2

Use the project as the source of truth for planned product work.

## Expectations

- Take ordinary roadmap tasks from the GitHub Project.
- Use the `Sequence` field as the intended roadmap order unless the project
  owner explicitly changes priority.
- Keep `Status`, `Type`, `Area`, `Priority`, and `Release` fields current.
- Leave unstarted work in `Todo`.
- Move work to `In Progress` when implementation starts.
- Move work to `Done` after the related pull request is merged or the issue is
  otherwise completed.
- Keep project issue titles, labels, bodies, comments, and metadata in English.
